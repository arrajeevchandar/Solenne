# Solenne Flutter Frontend

Flutter mobile frontend for the Solenne video journal flow: splash, onboarding,
Firebase Auth, home dashboard, journals, video recording, Cloudinary upload, and
Firestore journal metadata.

## Required configuration

Android Firebase is initialized from `android/app/google-services.json`.
The registered Android package is `com.spd.solenne`.
Cloudinary still uses unsigned uploads for this frontend prototype.

Run on Android with Cloudinary values:

```powershell
flutter run `
  --dart-define=CLOUDINARY_CLOUD_NAME=your-cloud-name `
  --dart-define=CLOUDINARY_UPLOAD_PRESET=your-unsigned-upload-preset
```

Or use the helper script:

```powershell
.\scripts\run_android.ps1 `
  -CloudinaryCloudName your-cloud-name `
  -CloudinaryUploadPreset your-unsigned-upload-preset
```

For web/iOS, `lib/firebase_options.dart` still expects Firebase dart-defines or
generated platform config.

## Debug build speed

The first Android run can be slow because Gradle builds native Firebase, camera,
permission, and video player plugins. After the first install, keep the
`flutter run` session open and use hot reload (`r`) instead of stopping and
reinstalling for every UI change. Avoid `flutter clean` unless Gradle cache
state is genuinely broken.

## Firestore paths

- `users/{uid}`
- `users/{uid}/journals/{journalId}`

## Cloudinary note

Unsigned uploads are suitable for this prototype only. Production should replace
them with signed parameters generated by a trusted backend or Firebase Function.
